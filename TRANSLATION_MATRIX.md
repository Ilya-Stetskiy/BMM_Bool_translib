# Матрица трансляций bmm-translib

Все 20 функций перевода между пятью представлениями булевых функций (AIG, BDD,
ANF, THR, TruthTable). Для каждой — вердикт «параллельно vs однопоточно» по
факту честных замеров (не оценка), реальная граница по числу переменных, и
обходной путь, если он существует и быстрее прямого.

Легенда вердикта:
- 🟢 параллелизм реально помогает (честный, воспроизводимый speedup)
- ⚪ последовательно по правилу конвенций (Bdd → только Sylvan/Lace, `core/CONVENTIONS.md` §6 правило 1)
- 🟡 параллелизм пробовали — устойчивого выигрыша нет (архитектурное ограничение)
- 🔴 жёсткая стена по n или риск краха

Источники цифр: `aig/README.md`, `bdd/README.md`, `anf/README.md`,
`thr/README.md`, `benchmarks/LARGE_SCALE_REPORT.md`,
`verify/CHAIN_TESTS_REPORT.md`.

## AIG →

### AIG → BDD (`aig_to_bdd`)
- Вердикт: ⚪ последовательно (Sylvan/Lace, правило 1)
- Предел по n: нет жёсткого предела — n=60 подтверждено (EPFL router, FORCE-эвристика)
- Обходной путь: —

### AIG → ANF (`aig_to_anf`)
- Вердикт: 🟡 TBB требуется правилом, но не используется (BRiAl не потокобезопасен — подтверждено ThreadSanitizer)
- Предел по n: нет предела по n — ограничение по времени (10с); структурный взрыв ANF не зависит от n (падал уже на 317 гейтах)
- Обходной путь: —

### AIG → THR (`aig_to_thr`)
- Вердикт: 🟢 TBB — честный растущий speedup 0.94→1.14→1.53x (n=8→12→14)
- Предел по n: ILP без жёсткого предела; риск — до 16М ограничений при n≈24 (не проверено эмпирически)
- Обходной путь: —

### AIG → TT (`aig_to_tt`)
- Вердикт: 🟡 OpenMP, allocation-free версия (2 фикса за сессию) — абсолютное время лучше, speedup всё ещё ~0.86–1.0x
- Предел по n: n ≤ 24 (`kMaxTruthTableVars`)
- Обходной путь: —

## BDD →

### BDD → AIG (`bdd_to_aig`)
- Вердикт: ⚪ последовательно (Sylvan/Lace, правило 1) — эталонное качество
- Предел по n: масштабируется отлично, тот же диапазон, что и `aig_to_bdd`
- Обходной путь: —

### BDD → ANF (`bdd_to_anf`)
- Вердикт: ⚪ последовательно (Sylvan/Lace, правило 1) — гибрид Дэвио + Möbius-fallback
- Предел по n: n ≤ 24 без BRiAl (fallback идёт через полную TT); с BRiAl — не найдено предела
- Обходной путь: —

### BDD → THR (`bdd_to_thr`)
- Вердикт: 🔴 без параллелизма (табличный lookup + верификация)
- Предел по n: прямой путь — жёстко K ≤ 6 (`CHOW_DATABASE`, 1111 записей); K>6 → `NotImplemented`
- Обходной путь: **`Bdd→Tt→Thr` работает для n=7…24**: 8.9 / 37.7 / 151.6 / 417мс (n=7/9/11/13)

### BDD → TT (`bdd_to_tt`)
- Вердикт: ⚪ последовательно (Sylvan/Lace, правило 1) — Θ(2^n), 3 итерации оптимизации
- Предел по n: n ≤ 24 (`kMaxTruthTableVars`)
- Обходной путь: —

## ANF →

### ANF → AIG (`anf_to_aig`)
- Вердикт: 🟡 TBB пробовали дважды независимо (в т.ч. отдельно студентом на своей ветке) — устойчивого выигрыша нет (+150%…−40%)
- Предел по n: нет жёсткого предела — стабильно работает до n=64 в корпусе
- Обходной путь: —

### ANF → BDD (`anf_to_bdd`)
- Вердикт: ⚪ последовательно (Sylvan/Lace, правило 1)
- Предел по n: риск краха (`abort()`) на bent/IP-функциях с «далёкими» переменными — уже при n≈44
- Обходной путь: **обход через `Tt` в 6–12× быстрее прямого пути** (n=11: 9.7x; n=17: 12.0x)

### ANF → THR (`anf_to_thr`)
- Вердикт: 🟡 TBB не добавлен сознательно — стоимость целиком в ILP-solve
- Предел по n: код разрешает n ≤ 20, практический потолок n≈12–16 (n=20: 34с на одну точку)
- Обходной путь: —

### ANF → TT (`anf_to_tt`)
- Вердикт: 🟡 OpenMP на Möbius-трансформе — без устойчивого выигрыша (bandwidth-bound)
- Предел по n: n ≤ 24, плотные функции резко замедляются к n≈20–24
- Обходной путь: —

---

## THR →

### THR → AIG (`thr_to_aig`)
- Вердикт: 🟡 TBB убран полностью в этой сессии (не побеждал ни разу, n=50…2000) — абсолютное время ×4.8 быстрее после удаления
- Предел по n: нет предела; реальный корпус n=51 — 1235 гейтов, 0.35–1.6мс
- Обходной путь: —

### THR → BDD (`thr_to_bdd`)
- Вердикт: ⚪ последовательно (Sylvan/Lace, правило 1) — DP с bounding
- Предел по n: реальный корпус n=51 — 6720 узлов, полиномиально
- Обходной путь: —

### THR → ANF (`thr_to_anf`)
- Вердикт: 🟢 OpenMP — реальный выигрыш после 2 найденных и исправленных перф-багов
- Предел по n: n ≤ 24 (построение TT ограничено памятью плоского массива)
- Обходной путь: —

### THR → TT (`thr_to_tt`)
- Вердикт: 🟢 OpenMP — растущий speedup: 1.0x→0.9x→1.4x→2.1x (n=12→16→20→24)
- Предел по n: n ≤ 24 (`kMaxTruthTableVars`)
- Обходной путь: —

## TT →

### TT → AIG (`tt_to_aig`)
- Вердикт: 🟢 TBB, мьютекс-фикс этой сессии — сильный speedup 9–13x при n=16 на случайных TT
- Предел по n: n ≤ 24 формально; на случайных (не majority-подобных) TT практически много меньше — риск взрыва узлов
- Обходной путь: —

### TT → BDD (`tt_to_bdd`)
- Вердикт: ⚪ последовательно (Sylvan/Lace, правило 1) — разложение Шеннона
- Предел по n: n ≤ 24 (`kMaxTruthTableVars`)
- Обходной путь: —

### TT → ANF (`tt_to_anf`)
- Вердикт: 🟡 OpenMP на трансформе — без выигрыша (сборка полинома BRiAl доминирует: 10.56 из 10.6с при n=22)
- Предел по n: n ≤ 24 формально; плотные функции — уже секунды на n≈20–22
- Обходной путь: —

### TT → THR (`tt_to_thr`)
- Вердикт: 🟡 ILP-solve сам по себе не параллелится; в этой сессии добавлен **предфильтр по унитарности** — отклоняет заведомо непороговые до дорогого ILP
- Предел по n: код разрешает n < 32, практический потолок n≈16 на реальных весах
- Обходной путь: —
- **Замер (n=12, неунитонный случайный вход, полный отказ)**: было 115.5мс (полный ILP до `INFEASIBLE`) → стало 47.1мс (предфильтр) — **в 2.45× быстрее**
